# 个人自媒体包装规划文档

---

## 1. 人设定位

### 一句话
一个提前毕业路线中的大学生 builder，用 AI 多智能体工作流把剧本、动画、游戏和系统从想法推到可访问的部署，并且公开记录全过程。

### 三个标签
1. **大学生 builder**（不是“学习者”，是已经部署产品的开发者）
2. **AI 原生工作流工程师**（用 Claude / Codex / OpenCode 补齐技术栈，不主张从零硬写）
3. **创作者型开发者**（剧本、叙事卡牌、动画、游戏，产品里带叙事表达）

---

## 3. 内容 Pillar（固定栏目）

### Pillar A：项目实录（Build in Public）
- 记录从 idea → 设计 → 开发 → 部署 → 复盘的全过程。
- 适用：妙喵私教、美加墨世界杯造世界、献书游戏、TalentMatch、OPC/ECC 工作流。
- 核心钩子：不是讲“我学会了什么”，而是“我用 4 天/3 天/1 个周末把什么上线了”。

### Pillar B：AI 工作流拆解
- 主题：Claude Code、Codex CLI、OpenCode 的实战用法；多智能体编排；自动 git push / 记忆沉淀；token 节省技巧。
- 核心钩子：展示真实项目里的 prompt、工具链、失败案例和迭代过程。

### Pillar C：大学生 builder 生存指南
- 主题：提前毕业学分规划、字节跳动实习准备、面试、北京实习、多线程并行管理。
- 核心钩子：不灌鸡汤，只给可执行的 checklist、时间表和踩坑记录。

### Pillar D：创作者的技术账本
- 主题：剧本、叙事卡牌、动画、游戏、音效、Lottie、AI 视频生成。
- 核心钩子：技术实现 + 审美/叙事表达，不是纯技术教程。

### Pillar E：数据与自动化执念
- 主题：爬虫、飞书 Bot、自动选题、自动分析、A/B 测试、自己给自己做增长工具。
- 核心钩子：把“手动操作”变成“自动决策/多智能体编排”的过程。

## 4. 内容形式与发布节奏

### 内容形式
- **短图文 / 笔记**：小红书、即刻。1-3 张图，配关键代码/部署截图/数据/时间线。
- **长文/复盘**：公众号、知乎。1000-2500 字，包含背景、过程、关键决策、结果、下一步。
- **短视频 demo**：小红书 / B站 / 抖音。15-60 秒，展示产品界面、语音交互、动画效果。
- **Twitter thread**：把长文拆成 5-10 条 tweet，英文面向 international builder。
- **直播/语音房间**：可选，在即刻或 B站做“周末 builder 进度同步”。

### 发布节奏（建议以周为单位）
- 每周 2-3 条短内容（小红书/即刻）
- 每 1-2 周 1 篇长文或深度视频（公众号/知乎/B站）
- 每次项目有重大更新（部署、新功能、比赛提交）同步 1 条短内容 + 1 条长文
- 每月 1 次“月度 builder 总结”，汇总所有项目进展与下个月目标

### 内容生产原则
- 70% 基于真实项目进展，30% 做回顾/拆解/方法提炼。
- 不为了发而发，没有进展就不发；有进展时多发几条不同角度。
- 所有内容都保留 git/文档/截图证据，避免人设夸大。

## 5. 差异化卖点

1. **不是“从零学编程”博主，而是“用 AI 把工作流工业化”的 builder**
   - 市场上大量教程是“从零学 Python”。你的差异是：已经会用 Claude/Codex/多智能体把项目推到生产环境。

2. **项目密度高，且真实可访问**
   - 妙喵私教、TalentMatch（yorkteam.cn）、献书游戏（47.101.204.120）都是可点开的链接，不是 PPT。

3. **创作者 + 工程师的复合视角**
   - 既有 FastAPI / React / XGBoost，也有剧本、叙事卡牌、Lottie 动画、音效。这能吸引两类受众：想做作品的创作者 + 想补审美的工程师。

4. **公开失败的复盘**
   - 你已经经历过“假数据清洗”“SSH 推失败切 HTTPS”“API key 失效”“Codex 登录问题”。这些比成功案例更有信任感。

5. **多线程并行的系统方法**
   - 不是“我很努力”，而是展示“我如何用工具、记忆沉淀、自动化和优先级管理多个项目”。

## 7. 风险与避坑

### 风险 1：人设过度承诺
- **表现**：提前毕业、字节实习、创业、援助非洲同时挂在嘴边，容易显得空洞。
- **避坑**：每个阶段只突出“当下在做且已推进到具体节点”的目标，其他目标作为“远期方向”一带而过。

### 风险 2：只晒技术，没有故事
- **表现**：满屏代码、部署图、通过率，用户看不懂也记不住。
- **避坑**：每条内容都要回答“为什么做、遇到什么、怎么解决、学到什么”，用故事承载技术。

### 风险 3：多线程失控导致断更
- **表现**：项目一多，内容节奏乱，账号变“项目日志堆砌”。
- **避坑**：建立选题库和发布日历，每次项目更新只取“一个最有价值的点”发内容，不要记流水账。

### 风险 4：流量焦虑走捷径
- **表现**：买课、刷量、过度标题党、制造焦虑。
- **避坑**：不靠课程变现，不靠刷量，标题可以吸引人但内容必须真实可查。

### 风险 5：泄露敏感数据或隐私
- **表现**：TalentMatch 涉及真实简历、飞书 Bot、候选人信息，稍不注意就会踩合规红线。
- **避坑**：所有截图必须脱敏；不展示真实姓名、电话、简历内容；服务器 IP/域名可公开但内部接口不公开。

### 风险 6：AI 生成内容未标注
- **表现**：用 AI 生成图片/视频/文案但不说，可能引发信任危机。
- **避坑**：主动说明“这部分由 AI 生成/辅助生成”，反而增强“AI 原生工作流”人设。

### 风险 7：忽视学业与实习主线
- **表现**：自媒体占用过多时间，影响学分、面试、实习准备。
- **避坑**：自媒体内容 70% 来自项目本身，减少“为了内容而做的内容”；把发内容当成项目记录的副产品。

## 8. 可立即执行的第一步

### 今天就能做的 3 件事

1. **用妙喵私教录一条 30 秒 demo 视频**
   - 打开宠物助手，点击麦克风输入，展示语音输入 → AI 回复 → TTS 朗读 → Lottie 动画反馈。
   - 平台：小红书 + 即刻（小红书发视频，即刻发动态/截图）。
   - 文案直接复用“第 1 条内容计划”的标题和钩子。

2. **建立自媒体选题库**
   - 在飞书 Doc 或 Notion 里建一页，把“内容 Pillar”和“前 10 条内容计划”写进去。
   - 每完成一个项目节点，立刻往库里丢一条备选选题，避免临时想选题。

3. **设定“项目更新 → 内容同步”触发规则**
   - 每次 git push / 部署 / 功能通过测试后，强制自己写 50 字以内的“一句话进展”发到即刻。
   - 每周日晚上花 30 分钟，把当周进展整理成 1 条小红书 + 1 条公众号/知乎备选。

### 本周目标
- 发布至少 2 条内容（1 条视频 + 1 条即刻/小红书图文）。
- 不追求完美，先验证“哪些项目进展能引起互动”，再优化选题和形式。

---

**总结**：你的自媒体不是“个人 IP 包装”，而是把你已经在做的事情公开化、结构化、可搜索化。核心资产是真实的项目、真实的失败、真实的进度。坚持这一点，流量只是副产品。
